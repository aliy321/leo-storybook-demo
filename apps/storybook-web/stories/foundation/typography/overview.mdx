# Typography

LEO typography ensures readability, hierarchy, and consistency across web and native. Sizes, weights, and line heights come from Figma tokens — not ad hoc pixel values.

See [Default font](/docs/foundation-typography-default-font--docs) and [Display font](/docs/foundation-typography-display-font--docs) for family-specific guidance.

## Typography color tokens

Text color roles adapt to brand and scheme. Use with size and weight utilities:

| Token | Class | Use |
| ----- | ----- | --- |
| Display | `text-display` | Large headlines |
| Heading | `text-heading` | Section titles |
| Paragraph | `text-paragraph` | Body content |
| Footnote | `text-footnote` | Captions and small text |

## Best practices

### Do

- Use token sizes (`text-14`, `text-24`) — combine with `font-*` and `leading-*`
- Pair `leading-14` with body-sized text for readability
- Reserve `font-mr-banks` for display and marketing moments
- Use `font-public-sans-pro` for all interface text
- Limit the number of sizes on a single screen

### Don't

- Don't use arbitrary font sizes outside the scale
- Don't mix font families without a clear hierarchy reason
- Don't use display font for dense UI like tables or forms
