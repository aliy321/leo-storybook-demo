# Spacing

The LEO spacing scale creates consistent visual rhythm across padding, margin, gap, border radius, and border width. Token keys use the `leo-` namespace (for example `p-leo-16` = 16px padding).

---

## Padding

Padding creates space **inside** an element.

<table>
  <thead>
    <tr><th>Class pattern</th><th>Description</th></tr>
  </thead>
  <tbody>
    <tr><td><code>p-{'{size}'}</code></td><td>All sides</td></tr>
    <tr><td><code>px-{'{size}'}</code></td><td>Horizontal (left + right)</td></tr>
    <tr><td><code>py-{'{size}'}</code></td><td>Vertical (top + bottom)</td></tr>
    <tr><td><code>pt-{'{size}'}</code>, <code>pr-{'{size}'}</code>, <code>pb-{'{size}'}</code>, <code>pl-{'{size}'}</code></td><td>Individual sides</td></tr>
  </tbody>
</table>

Replace `{size}` with a scale token (`leo-4`, `leo-16`, etc.).


## Gap

Gap applies consistent spacing between flex or grid children.

<table>
  <thead>
    <tr><th>Class</th><th>Description</th></tr>
  </thead>
  <tbody>
    <tr><td><code>gap-{'{size}'}</code></td><td>Gap between all children</td></tr>
    <tr><td><code>gap-x-{'{size}'}</code></td><td>Horizontal gap only</td></tr>
    <tr><td><code>gap-y-{'{size}'}</code></td><td>Vertical gap only</td></tr>
  </tbody>
</table>

```tsx
<View className="flex-row gap-leo-12">
  <Button>Cancel</Button>
  <Button>Save</Button>
</View>
```

## Best practices

### Do

- Use scale values consistently — `p-leo-16` for standard card padding, `gap-leo-8` for tight groups
- Increase spacing between unrelated groups; decrease within a group
- Use `gap-*` in flex layouts instead of margin on every child

### Don't

- Don't use arbitrary pixel values outside the token scale
- Don't mix incompatible spacing rhythms on the same screen without intent
