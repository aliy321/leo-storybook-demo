# Shadows

LEO provides **theme-aware** elevation tokens that create depth and visual hierarchy. Shadow values switch automatically between light and dark schemes via CSS variables set by `data-color-scheme` (web) or `Theme` (native).

Toggle **Scheme** in the Storybook toolbar to compare light and dark shadow values.

## Usage

### Web and native (utility classes)

```tsx
<View className="bg-surface shadow-sm rounded-8 p-leo-16">
  <Text>Card at rest</Text>
</View>

<View className="bg-surface shadow-lg rounded-8 p-leo-16">
  <Text>Modal panel</Text>
</View>
```

Components apply elevation through semantic `shadow-*` utilities in their platform CVA definitions.

---

## Best practices

### Do

- Use theme-aware `shadow-*` classes — values adapt to light and dark automatically
- Match shadow level to interaction importance (resting cards → `sm`, modals → `lg` or `xl`)
- Test elevation in both schemes before shipping

### Don't

- Don't hardcode `box-shadow` values in product code
- Don't mix arbitrary shadow levels on the same screen without hierarchy intent
- Don't use heavy shadows on flat, non-interactive surfaces
