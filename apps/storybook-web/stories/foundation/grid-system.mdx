# Grid system

The LEO grid defines how content aligns and responds across breakpoints. Layout tokens are generated from Figma and exported alongside the Tailwind theme (`gridConfig` in `@leo/tokens/rn`).

Design mobile-first, then add columns as viewport width increases.

## Usage

- Lay out pages with consistent columns and gutters from the grid spec
- Align components to grid columns rather than arbitrary widths
- Use token spacing (`p-leo-16`, `gap-leo-16`) for gutters and margins where utilities apply
- On native, prefer flex layouts that mirror column counts at each breakpoint

```tsx
// Web: responsive column layout with Tailwind breakpoints
<View className="flex-row flex-wrap gap-leo-16 tablet:gap-leo-16">
  <View className="w-full tablet:w-1/2 desktop:w-1/3">{/* column */}</View>
</View>
```

---

## Principles

- Mobile-first — design for 4 columns, then expand
- Keep content within the container width at each breakpoint
- Maintain 16px gutters consistently across breakpoints
- Increase side margins on tablet and desktop to frame content
