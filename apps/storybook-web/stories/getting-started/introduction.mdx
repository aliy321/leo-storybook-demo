# Getting started

**Leo Design System** is the single source of truth for styling, UI patterns, and front-end implementation across Great Eastern products.

**The design system consists of:**

- Foundation primitives (colors, typography, icons, spacing, elevation)
- Code-first design tokens compiled into web CSS, React Native themes, and Figma variables
- Platform-owned UI components with semantic `cva` variants
- Storybook docs for implementation and QA

## Monorepo layout

```text
packages/tokens   → Figma JSON → global.css + themes.css + RN theme
packages/web      → Stencil web-component package
packages/native   → React Native package
apps/storybook-*  → documentation and previews
```

## Quick start

From the repo root:

```bash
pnpm install
pnpm dev:web    # web Storybook on :6006
pnpm dev:rn     # native Storybook on :6007
```

## Using tokens

Import global tokens once at the app entry:

```css
@import '@leo/tokens/css';
```

Apply brand and color scheme on the document root:

```html
<html data-brand="default" data-color-scheme="light">
```

See [Design token](/docs/foundation-design-token--docs) for naming and usage.

## Using components

Implementations and their platform CVA definitions live in `@leo/web` and `@leo/native`. Start from component stories under **Components** in the sidebar.

> Keep semantic intent aligned, but let each platform express its own primitive states and size API.

## Do's and Don'ts

### Do's

- Use design tokens for color, spacing, and typography
- Compose screens from documented components
- Check Foundation pages before introducing new visual patterns

### Don'ts

- Do not hardcode foundation values in product code
- Do not override component styling in consuming apps — request changes in the design system instead
- Do not mix third-party UI libraries for patterns already covered by LEO
