# Button

Button triggers an action. For navigation, use a link.

**Supported screens:** Mobile, Tablet, Desktop

## Usage

- Use **default** for the main action in a section.
- Use **secondary** for supporting actions.
- Use **outline** or **ghost** for less prominent actions.
- Use **destructive** only for actions that remove or damage data.
- Keep labels short and action-led, like "Save changes" or "Add item".

## Anatomy

- Root: press target, layout, background, border, focus, and disabled state.
- Text: label color, size, weight, and hover/active text state.
- Icon slots: optional leading and trailing affordances.

## Accessibility

- Button text and background colors must meet WCAG contrast requirements.
- Icon-only buttons must include an accessible label via `aria-label` or visible text.
- Disabled buttons should not be the only path to complete a task.

## Implementation

Button follows the CVA + Reusables pattern: root and text styles are separate CVA recipes, shared by web and React Native. Recipes use normal Tailwind v3 utilities, while design tokens provide semantic color and theme values.
