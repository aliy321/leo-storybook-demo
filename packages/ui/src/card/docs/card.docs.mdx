# Card

Card groups related content in a contained surface.

**Supported screens:** Mobile, Tablet, Desktop

## Usage

- Use **elevated** for primary content blocks with subtle shadow.
- Use **outline** when you need a clear boundary without elevation.
- Use **filled** for lower-emphasis grouped content on the page background.
- Set `hasPadding={false}` when child content manages its own spacing.

## Accessibility

- Cards are presentational unless made interactive.
- When clickable, provide context in slotted content or an accessible name.
- Disabled cards must not trigger actions.

## Implementation

Card follows the CVA + semantic token pattern shared with Button. Variants are defined once in `packages/ui/src/card/core/card.core.ts` and consumed by web (Stencil) and native (React Native) shells.
